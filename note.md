This program implements a minimal version of word2vec in pure NumPy using the skip-gram architecture with negative sampling. It starts by reading a text corpus from a file and tokenizing it into lowercase word tokens. Then it builds a vocabulary, counts word frequencies, and maps each word to an integer id. After that, it converts the whole corpus into a sequence of ids so the model can work with numbers instead of strings. The function generate_pairs() creates positive skip-gram training pairs by taking each center word and pairing it with words inside a fixed context window. The function build_negative_distribution() prepares a probability distribution for negative sampling based on word frequencies raised to the power of 0.75. For each positive pair, sample_negatives() draws several negative word ids that act as false context examples. The Word2Vec class stores two embedding matrices: one for center words and one for context words. In train_step(), the model computes dot-product scores for the positive and negative pairs, applies the sigmoid function, and computes the skip-gram with negative sampling loss. It then manually computes gradients for the embeddings and updates only the rows involved in the current sample using SGD. The train() function organizes the full training process by preparing the data, looping over epochs, sampling negatives, calling train_step(), and printing the average loss per epoch. After training, the program uses the input embeddings as the learned word representations. The function nearest_neighbors() normalizes the embeddings and finds the most similar words to a given query word using cosine-style similarity. Finally, main() loads the corpus, trains the model, and prints nearest neighbors for a few example words as a simple qualitative evaluation.
